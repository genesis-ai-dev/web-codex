Getting Started with Codex Web

Create your first cloud development workspace in minutes

Quick Start

Get up and running with Codex Web in 3 simple steps:

1

Sign Up

Create your account using Google OAuth or AWS Cognito. No credit card required for the free trial.

Create Account
2

Create a Workspace

Choose your resource tier and create your first development workspace.

curl -X POST https://api.codexweb.dev/api/workspaces \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-first-workspace",
    "tier": "team",
    "groupId": "YOUR_GROUP_ID"
  }'
3

Start Coding

Access your workspace through the web interface or connect via SSH. Your workspace will be ready in under 30 seconds.

Installation

Install the Codex CLI for easier workspace management:

npm install -g @codex/cli
yarn global add @codex/cli
brew install codex-cli

Verify the installation:

codex --version
# codex version 1.0.0

Authentication

Codex Web supports multiple authentication providers:

Supported Providers

Google OAuth, AWS Cognito, or email/password authentication

Using the CLI

codex login

# Follow the browser prompt to authenticate
# Your token will be stored securely in ~/.codex/credentials

Using API Tokens

For programmatic access, generate an API token from your dashboard:

export CODEX_API_KEY="your-api-key-here"

# Use in API requests
curl -H "Authorization: Bearer $CODEX_API_KEY" \
  https://api.codexweb.dev/api/workspaces

Creating Your First Workspace

Workspaces are isolated development environments running in Kubernetes containers.

Using the CLI

codex workspace create my-project \
  --tier team \
  --template node-typescript

# Workspace created successfully!
# URL: https://my-project-abc123.codexweb.dev
# Status: Running

Using the API

POST /api/workspaces
Content-Type: application/json
Authorization: Bearer YOUR_TOKEN

{
  "name": "my-project",
  "tier": "team",
  "groupId": "group-123",
  "template": "node-typescript"
}
Workspace Ready!

Your workspace is now accessible at the provided URL. You can start coding immediately.

Understanding Workspaces

Workspaces are the core of Codex Web. Each workspace provides:

  • Isolated development environment
  • Dedicated compute resources (CPU, RAM)
  • Persistent storage for your code
  • Pre-configured development tools
  • Web-based IDE access
  • SSH and VS Code Remote support

Workspace Actions

Action Description CLI Command
start Start a stopped workspace codex workspace start <id>
stop Stop a running workspace codex workspace stop <id>
restart Restart a workspace codex workspace restart <id>
delete Permanently delete a workspace codex workspace delete <id>

Resource Tiers

Choose the right resource tier for your workload:

Tier CPU Memory Storage Best For
Single User 1 core 2GB 20GB Individual projects, learning
Small Team 2 cores 4GB 20GB Team collaboration, medium projects
Enterprise Custom Custom Custom Large-scale applications, high-performance needs
Resource Management

CPU requests are guaranteed, but no limits are set to allow burst performance. Memory limits equal requests for consistency.

Workspaces API Reference

List Workspaces

GET /api/workspaces

Returns all workspaces for the authenticated user.

Create Workspace

POST /api/workspaces

Creates a new workspace with the specified configuration.

Request Body

{
  "name": "string",           // Required: Workspace name
  "tier": "string",           // Optional: single|team|enterprise
  "groupId": "string",        // Required: Group ID
  "template": "string",       // Optional: Template name
  "resources": {              // Optional: Custom resources
    "cpu": "2",
    "memory": "4Gi",
    "storage": "20Gi"
  }
}

Get Workspace

GET /api/workspaces/:id

Update Workspace

PUT /api/workspaces/:id

Delete Workspace

DELETE /api/workspaces/:id

Workspace Actions

POST /api/workspaces/:id/actions 
{
  "action": "start" | "stop" | "restart"
}

Error Handling

The API uses standard HTTP status codes and returns errors in this format:

{
  "error": {
    "code": "WORKSPACE_NOT_FOUND",
    "message": "Workspace with ID 'abc123' not found",
    "statusCode": 404
  }
}

Common Error Codes

Code Status Description
UNAUTHORIZED 401 Invalid or missing authentication token
FORBIDDEN 403 Insufficient permissions for this resource
NOT_FOUND 404 Resource not found
VALIDATION_ERROR 400 Invalid request parameters
RATE_LIMIT_EXCEEDED 429 Too many requests
